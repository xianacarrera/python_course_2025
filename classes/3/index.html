Escritura de archivos (output)

La funci√≥n open() tiene varios modos para abrir ficheros:

  • r -> Read mode. Lee el archivo desde el inicio. Si el archivo no existe, se genera un error.
  • w -> Write mode. Borra el contenido del archivo y permite escribir nuevo texto sobre √©l. Si el archivo no existe, lo crea.
  • a -> Append mode. A√±ade nuevo texto desde el final del archivo. Si el archivo no existe, lo crea.
  • x -> Create mode. Crea el archivo y permite escribir en √©l. Si ya existe, genera un error.

"r" es la opci√≥n por defecto: open("fichero.txt", "r") y open("fichero.txt") hacen lo mismo.

A cada uno de estos modos se le puede a√±adir "b", para manejar los archivos en binario, o "t", para manejarlos como archivos de texto.

"t" es la opci√≥n por defecto, as√≠ que no es necesario indicarla.

Si queremos leer y escribir a la vez, podemos usar "r+", "w+", "a+" o "x+" (¬°que no son iguales!).

Funciones lambda

  • Son funciones an√≥nimas (i.e., sin nombre) √∫tiles para tareas peque√±as y sencillas.
  • Su origen est√° en la investigaci√≥n sobre el concepto de funci√≥n que dio origen al c√°lculo lambda de Alonzo Church (1930s).
  • El estilo funcional del c√°lculo lambda y las funciones lambda busca actuar sin tener en cuenta el estado actual del sistema y evitar los efectos secundarios.
  • Python no es un lenguaje funcional, pero adopt√≥ conceptos de lenguajes funcionales.
  • Las funciones lambda solo pueden tener expresiones puras: no puede haber expresiones con asignaciones (=), while, try, etc.
Unlike lambda forms in other languages, where they add functionality, Python lambdas are only a shorthand notation if you‚Äôre too lazy to define a function. A√±adir fuente
Lambda calculus es Turing completo (cualquier tipo de c√°lculo, equiv a M√°quina de Turing). Los lenguajes funcionales utilizan la filosof√≠a del c√°lculo lambda. Lenguajes funcionales: Haskell, Lisp, Erlang... Yes, son sintactic sugar.
