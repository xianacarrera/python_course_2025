Excepciones

Si durante la ejecuci√≥n ocurre un error, el programa finalizar√° abruptamente.

El manejo de excepciones nos permite indicar al programa c√≥mo afrontar un error de manera controlada para que pueda continuar ejecut√°ndose.

Los bloques de manejo de excepciones utilizan 4 keywords (ninguna obligatoria):

  • try: intenta hacer esto, a ver si no falla $\dots$
  • except: si falla algo dentro del try, haz esto $\dots$
  • else: si todo va bien dentro del try, haz esto $\dots$
  • finally: pase lo que que pase, haz esto $\dots$

        # C√≥digo que puede producir un error
        pass
    except:

        # El resto de secuencias de try no se ejecutan
        pass
    else:
        # Se ejecuta despu√©s de todo el c√≥digo de try si no se produce un error
        pass
    finally:
        # Se ejecuta en √∫ltimo lugar, independientemente de si se ha producido un error o no
        pass

Nota: el orden es importante. try-else-except no es v√°lido. Quitar los pass y preguntar por qu√© el c√≥digo dar√≠a un error? else es √∫til para lo que dependa de que el bloque de try tenga √©xito else y finally son anecd√≥ticos Puede haber else sin except, finally sin ... pero todos dependen de try

Escritura de archivos (output)

La funci√≥n open("archivo.txt", modo) tiene varios modos para abrir ficheros:

  • r $\rightarrow$ Read mode. Lee el archivo desde el inicio. Si el archivo no existe, se genera un error.
  • w $\rightarrow$ Write mode. Borra el contenido del archivo y permite escribir nuevo texto sobre √©l. Si el archivo no existe, lo crea.
  • a $\rightarrow$ Append mode. A√±ade nuevo texto desde el final del archivo. Si el archivo no existe, lo crea.
  • x $\rightarrow$ Create mode. Crea el archivo y permite escribir en √©l. Si ya existe, genera un error.

r es la opci√≥n por defecto: open("fichero.txt", "r") y open("fichero.txt") hacen lo mismo.

A cada uno de estos modos se le puede a√±adir b, para manejar los archivos en binario, o t, para manejarlos como archivos de texto (por defecto).

Funciones lambda

Funciones lambda

  • Son funciones an√≥nimas (i.e. sin nombre) √∫tiles para tareas peque√±as y sencillas.
  • Su origen est√° en la investigaci√≥n sobre el concepto de funci√≥n que dio origen al c√°lculo lambda de Alonzo Church (1930s).
  • El estilo funcional del c√°lculo lambda y las funciones lambda busca actuar sin tener en cuenta el estado actual del sistema y evitar los efectos secundarios.
  • Python no es un lenguaje funcional, pero adopt√≥ algunos conceptos de lenguajes funcionales.
  • Las funciones lambda solo pueden tener expresiones puras. No puede haber expresiones con asignaciones $=$, ni bloques while, try, etc.
Unlike lambda forms in other languages, where they add functionality, Python lambdas are only a shorthand notation if you're too lazy to define a function.
Lambda calculus es Turing completo (cualquier tipo de c√°lculo, equiv a M√°quina de Turing). Los lenguajes funcionales utilizan la filosof√≠a del c√°lculo lambda. Lenguajes funcionales: Haskell, Lisp, Erlang... Yes, son sintactic sugar.

  • lambda indica que se trata de una funci√≥n lambda
  • argumentos: Los par√°metros (0, 1 o varios). Estas variables solo pueden ser usadas dentro de la lambda en la que se declaran.
  • expresi√≥n: El cuerpo de la funci√≥n, que indica qu√© devuelve. Debe ser una √∫nica expresi√≥n.

Es un paradigma de programaci√≥n basado en el concepto de objetos.
